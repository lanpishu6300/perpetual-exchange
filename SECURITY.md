# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue. Instead, please email security@[your-domain].com with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and work with you to resolve the issue before public disclosure.

## Security Best Practices

- Keep dependencies up to date
- Use secure configuration defaults
- Enable authentication and authorization
- Monitor for suspicious activity
- Regular security audits
