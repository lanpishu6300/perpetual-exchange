# Contributing Guide

Thank you for your interest in contributing to Perpetual Exchange!

## Language Guidelines

This project supports internationalization with English as the primary language and Chinese as a secondary language.

### Documentation

- All new documentation should be written in English first
- Chinese translations (`.zh-CN.md`) are welcome but optional
- Key documents should have both English and Chinese versions:
  - `README.md` / `README.zh-CN.md`
  - `BENCHMARK_GUIDE.md` / `BENCHMARK_GUIDE.zh-CN.md`
  - `docs/README.md` / `docs/README.zh-CN.md`

### Code

- Code comments should be in English
- Variable names, function names, and class names should be in English
- Error messages can be in English (or use i18n if implemented)

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add tests if applicable
5. Update documentation (English + Chinese if possible)
6. Submit a pull request

## Code Style

- Follow existing code style
- Use meaningful variable names
- Add comments for complex logic
- Keep functions focused and small

## Questions?

Feel free to open an issue for questions or discussions.
